## 🛠️ 실전! 필수 Git 명령어 모음

코드 작업부터 저장소 반영까지, 매일 사용하는 핵심 명령어들입니다.

### 1. 상태 확인 및 추가
작업한 코드를 Git이 인식하도록 준비하는 단계입니다.
* `git status`: 현재 내 프로젝트의 상태(수정된 파일, 스테이징된 파일 등)를 확인합니다.
* `git add .`: 수정된 모든 파일을 한 번에 스테이징 영역(업로드 대기 상태)에 추가합니다.
* `git add [파일명]`: 특정 파일만 골라서 스테이징합니다.

### 2. 변경 내용 확정 (커밋)
수정한 내용에 의미 있는 이름을 붙여 기록하는 단계입니다.
* `git commit -m "메시지"`: 변경 사항을 확정합니다. 
    * *팁: "메시지" 부분에는 '로그인 기능 구현', '메인 화면 디자인 수정' 등 무엇을 수정했는지 명확하게 적어주세요.*

### 3. 원격 저장소 동기화 (GitHub 반영)
내 PC의 기록을 GitHub 서버로 올리거나, 서버의 최신 코드를 내 PC로 가져옵니다.
* `git push origin [브랜치명]`: 내 로컬 컴퓨터의 커밋 내역을 GitHub 원격 저장소에 올립니다.
* `git pull origin [브랜치명]`: GitHub 원격 저장소의 최신 코드를 내 컴퓨터로 내려받아 동기화합니다.

### 4. 브랜치 관리
협업 시 기능별로 코드를 분리하는 핵심 명령어입니다.
* `git branch`: 현재 생성된 브랜치 목록과 내가 위치한 브랜치를 확인합니다.
* `git checkout [브랜치명]`: 해당 브랜치로 이동합니다.
* `git checkout -b [브랜치명]`: 새 브랜치를 생성함과 동시에 그 브랜치로 이동합니다.

---

### 💡 [참고] 명령어 사용 순서 요약 (Workflow)

```bash
# 1. 현재 프로젝트 변경 상태 확인 (어떤 파일이 수정되었는지 확인)
git status
# 2. 수정된 모든 파일을 업로드 대기 상태(Staging Area)로 추가
git add .
# 3. 변경 사항 최종 확정 및 의미 있는 메시지 기록
git commit -m "수정 내용 요약 (예: 로그인 기능 구현)"
# 4. 내 PC의 기록을 GitHub 원격 저장소로 업로드
git push origin main
